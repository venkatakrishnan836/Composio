Step 3 Common Blockages Analyzed

No Public Developer API (16 Apps)

The platform is strictly consumer-facing or does not provide an external endpoint catalog. Integrators would have to rely on complex browser-automation scripts or RPA rather than direct API calls.

CLI-Only / Local Utility (5 Apps)

These tools (like local code formatters or OS utility CLI wrappers) run locally in a console environment. They have no cloud auth methods or hosted API endpoints, requiring agent code execution sandboxes.

Gated Access Models

Even for buildable tools, authentication models create barriers: 16 apps require sales team interaction, custom enterprise plans, or specialized partner approvals before developer credentials are issued.

Step 4 The Research Agent Architecture

Pass A — Web Discovery

Scrapes web documentation for the target app:

  • Search query: "{app} API docs developer reference"
  • Crawls the top URL using Playwright for rich dynamically rendered content
  • Extracts key features using LLaMA-3.1-8B models

Pass B — Registry Check

Cross-references verified tools in SDK registry:

  • Queries backend Composio toolkit registry
  • Extracts active auth schemas directly (acting as baseline ground truth)
  • Retrieves direct official developer site configuration parameters

Conflict Adjudication

Reconciles raw search discrepancies:

  • Auto-diff algorithm flags mismatches between Pass A and Pass B
  • Adjudicator LLM reads raw API metadata to make a precise final evaluation
  • Applies domain authority priority rules to override broken links

Where Human Was Needed

Human oversight is maintained for:

  • Resolving low-confidence items flagged by LLM as unverifiable
  • Detecting account-specific internal structures (e.g. LiveAgent support portals)
  • Manual validation tests against local CLI execution boundaries

💡 Key Insights & Pattern Discovery

🏆 Top Performers

  • Communications & Messaging — 100% buildable, 100% self-serve, but only 30% in SDK (massive opportunity)
  • Developer Infra & Data — 100% buildable, 100% self-serve, 80% already in SDK (mature integration surface)
  • Productivity & Project Management — 100% buildable, 80% in SDK (strong coverage)

Action: Prioritize Communications & Messaging for immediate SDK expansion — low friction, high return.

⚠️ High-Risk Categories

  • AI Research & Media — Only 40% self-serve, 60% gated. Just 10% in SDK.
  • Finance & Fintech — 70% self-serve, but only 50% buildable. Heavy gating and compliance barriers.
  • Ecommerce — 90% self-serve, 90% buildable, but only 20% in SDK (integration gap).

Action: Engage partnerships for AI Research apps. Consider alternative integration strategies (RPA, webhooks) for blocked platforms.

🔑 Pattern: The "Integration Gap"

  • Self-serve ≠ Buildable ≠ In SDK — Many apps are accessible but not yet integrated.
  • Ecommerce has a 70% gap between "Self-serve" (90%) and "In SDK" (20%).
  • Marketing & Social has a 40% gap (90% buildable, 60% in SDK).

Insight: The biggest opportunity lies in bridging the "Integration Gap" — especially in Ecommerce and Marketing categories.

📈 MCP-Capable Breakdown

  • 20 apps are MCP-capable — a key differentiator for AI agent tooling.
  • MCP is concentrated in Developer Infra (8 apps) and Communications (5 apps).
  • MCP-capable apps are 6× more likely to be in the "Easy Win" category.

Action: Prioritize MCP-capable apps for agent tooling — they provide the most value with the least development effort.

🎯

The Bottom Line

84 immediate wins (49 in SDK + 35 easy) represent 84% of all researched apps. The biggest unlock is bridging the Integration Gap — especially in Ecommerce (70% gap) and Marketing (40% gap) categories. Prioritize Communications & Messaging and Developer Infra for highest ROI.
